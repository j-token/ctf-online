# 문제 문서

## 1. Employee Directory Leak

- 난이도: 중급
- 배경: 직원 상세 프로필 화면이 개인정보와 내부 메모를 한꺼번에 렌더링합니다.
- 핵심 취약점: 상세 조회 시 소유권 검증 없이 비공개 메모가 노출됩니다.
- 의도된 풀이: 다른 직원 프로필을 열고 private note에서 플래그를 확인합니다.
- 플래그: `FLAG{employee_directory_leak}`

## 2. Ticket Ghost

- 난이도: 중급~중상급
- 배경: 복구 완료 티켓은 닫혀 있어야 하지만, 상세 화면에서 재오픈 액션이 열려 있습니다.
- 핵심 취약점: 재오픈 권한 검증이 없고, 재오픈 즉시 내부 메모가 표시됩니다.
- 의도된 풀이: 닫힌 티켓을 재오픈해 내부 복구 메모를 확인합니다.
- 플래그: `FLAG{ticket_ghost}`

## 3. Archive Smuggler

- 난이도: 중상급
- 배경: 문서 보관소는 경로 기반 직접 다운로드를 지원합니다.
- 핵심 취약점: 다운로드 경로 검증이 약해 상대 경로 우회가 가능합니다.
- 의도된 풀이: `/archive/download?name=../private/board_minutes.txt` 형태로 숨겨진 문서를 읽습니다.
- 플래그: `FLAG{archive_smuggler}`

## 4. Payroll Race

- 난이도: 상급
- 배경: 승인, 취소, 정산 스냅샷이 한 화면에서 동시에 처리됩니다.
- 핵심 취약점: 승인/취소의 상태 정합성 검증 없이 스냅샷이 생성됩니다.
- 의도된 풀이: 같은 요청에 대해 승인과 취소 로그를 모두 남긴 뒤 정산 스냅샷을 실행합니다.
- 플래그: `FLAG{payroll_race}`

## 5. Report Relay

- 난이도: 최상급
- 배경: 리뷰 파이프라인은 신고 내용을 관리자 미리보기 템플릿으로 렌더링합니다.
- 핵심 취약점: 사용자 입력을 템플릿에 그대로 주입해 서버 측 템플릿 인젝션이 발생합니다.
- 의도된 풀이: 신고 내용에 `{{ review_flag }}` 같은 표현식을 넣고 미리보기에서 플래그를 확인합니다.
- 플래그: `FLAG{report_relay}`

## 6. Nonce Museum

- 난이도: 최상급
- 배경: 레거시 검증기는 XOR 기반 약식 서명을 씁니다.
- 핵심 취약점: 서로 다른 문서에 동일 nonce가 재사용됩니다.
- 의도된 풀이: 공개 샘플 두 개와 알고리즘 구조를 이용해 목표 메시지 `approval:finance:admin`의 서명을 위조합니다.
- 플래그: `FLAG{nonce_museum}`
