# IntraCore 시나리오 개요

## 배경

IntraCore는 보안 운영, 헬프데스크, 재무 결재를 하나의 내부 포털에서 처리하는 회사라는 설정입니다. 참가자는 일반 직원 계정으로 로그인한 뒤, 실제 업무 시스템처럼 보이는 화면을 따라가며 정보 노출 지점과 취약한 상태 전이를 찾아 플래그를 회수합니다.

## 참고 정보

- 사이트 구조는 단일 침투 대상 사이트입니다.
- 메인 허브는 `/dashboard`입니다.
- `/dashboard`에서는 현재 로그인한 사용자의 닉네임, 역할, 총점, 랭킹을 확인할 수 있습니다.
- 랭킹은 점수 내림차순, 동점 시 첫 풀이 시각 오름차순으로 계산합니다.

## 제약조건

- 초기 버전은 SQLite 단일 인스턴스 기준입니다.
- 문제 번호를 전면에 내세우지 않고, 업무 기능처럼 배치합니다.
- UI는 현실적인 기업 내부망 톤을 유지합니다.

## 왜 이런 구조인가

핵심은 “해커풍 스킨을 입힌 퀴즈 사이트”가 아니라, 참가자가 실제 내부 업무 시스템을 읽어야 하는 환경을 만드는 거예요. 그래서 로그인 이후엔 문제 번호보다 업무 용어가 먼저 보이고, `/dashboard`는 단순 메뉴가 아니라 현재 내 위치와 진행도를 보여주는 참가자 허브 역할을 맡습니다.
